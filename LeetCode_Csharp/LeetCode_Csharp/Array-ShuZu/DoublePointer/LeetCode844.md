# 1. LeetCode844

除了在已经写好的 LeetCode844 已经写好的一种方法，时间复杂度和空间复杂度为 O(m+n)，当然用 StringBuilder 或者 stack 也是这种时空复杂度。

还有一种思想方法：就是从后往前（反向思考）：
同时反向遍历两个字符串，对于每个字符串，遍历过程中执行如下操作。

* 如果遇到退格键，则将删除字符个数加 1。

* 如果遇到非退格键且删除字符个数大于 0，则将删除字符个数减 1，表示当前字符被删除。

* 如果遇到非退格键且删除字符个数等于 0，则当前字符不被删除。

每次遍历到两个字符串的不被删除的字符或字符串首端时，判断两个字符串的对应字符是否存在和是否相等。

* 如果两个字符串的对应字符存在且不相等，则两个字符串输入之后的结果不相等，返回 false。

* 如果只有一个字符串的对应字符存在，则两个字符串输入之后的结果不相等，返回 false。

* 如果两个字符串都遍历到字符串首端，则两个字符串输入之后的结果相等，返回 true。
具体代码实现见 LeetCode844.cs
